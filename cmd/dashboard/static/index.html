⚙ Modo de operacion

Desarrollo: Errores detallados, certificados auto-firmados permitidos, TLS opcional. Produccion: Errores genéricos al cliente, TLS obligatorio, certificates validados.

Desarrollo: Puedes desactivar TLS para SQL Server 2008/2012 (no soportan TLS 1.2). Produccion: SIEMPRE activado. No se puede desactivar.

AVISO: MSSQL_ENCRYPT=false solo funciona con DEVELOPER_MODE=true

🔒 Seguridad

Activado: Solo SELECT permitido. Modificaciones (INSERT/UPDATE/DELETE/CREATE/DROP) bloqueadas excepto en tablas de la whitelist. Desactivado: Acceso completo a la base de datos.

Tablas y vistas que pueden ser modificadas en modo solo lectura. Separadas por comas. Ejemplo: temp_ai, v_temp_ia, mi_vista

Otras bases de datos en el mismo servidor que puedes consultar. Habilita: SELECT * FROM OtraBD.dbo.Tabla

Activado (default): Pide confirmacion antes de DROP TABLE, ALTER VIEW, TRUNCATE, etc. en objetos existentes. Desactivado: Ejecuta sin preguntar. Usa solo en CI/CD automatizado.

🤖 Modo Autopilot

Modo autonomo para desarrollo con IA. Omite confirmacion de operaciones destructivas Y validacion de schema. La whitelist sigue protegiendo. Ideal cuando la IA necesita operar sin interrupciones.

Autopilot activo:
  • Sin confirmacion para DDL destructivo
  • Sin validacion de schema
  • Whitelist sigue protegiendo
